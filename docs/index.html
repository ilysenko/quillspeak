Local Linux voice typing

Hold a trigger, speak, release, paste the result.

QuillSpeak is a Rust GTK4/libadwaita desktop app for local push-to-talk transcription with whisper.cpp models, a tray indicator, X11 hotkeys, Wayland-friendly trigger commands, scripts, clipboard output, and optional paste shortcuts.

Quick Start

  1. Install the latest .deb from GitHub Releases.
  2. Start QuillSpeak with quillspeak.
  3. Open Show Settings from the tray indicator.
  4. Download a ready whisper.cpp model from the Models page.
  5. Choose a shortcut profile, model, language, and output action.
  6. Hold the trigger, speak, release, and paste or copy the text.

Runtime Tools

Clipboard and paste output intentionally use external Linux tools so X11 and Wayland clients see the real system clipboard. 

sudo apt install wl-clipboard xclip xdotool ydotool

Speaker mute during recording uses PipeWire tools when that option is enabled. 

sudo apt install wireplumber pipewire-bin

Wayland

Use external hotkey tools with quillspeak trigger

Wayland compositors do not allow regular desktop apps to capture global keyboard shortcuts directly. QuillSpeak therefore keeps recording logic inside the app and exposes an explicit command mode that external hotkey tools can call on key down and key up. 

quillspeak trigger Default start
quillspeak trigger Default stop
quillspeak trigger Default toggle

The command talks to the running app through $XDG_RUNTIME_DIR/quillspeak/command.sock. Shortcut selectors resolve by exact id first and then by exact unique display name. Failed no-op commands exit non-zero so scripts can react.

Example swhkd configuration

ctrl + space
    quillspeak trigger Default start

ctrl + @space
    quillspeak trigger Default start

ctrl + shift + space
    quillspeak trigger Default stop

ctrl + shift + @space
    quillspeak trigger Default stop

Signal fallback

Linux signal triggers are available when a compositor or hotkey utility cannot run command-mode start/stop edges. Configure a shortcut as linux_signal, then send supported signals. 

pkill -USR1 -x quillspeak
pkill -USR2 -x quillspeak

Supported values are SIGUSR1, SIGUSR2, SIGALRM, and SIGWINCH. If the same signal is used for start and stop, the first signal starts recording and the next matching signal stops the active recording.

Tray state

  • White icon: idle.
  • Red icon: recording.
  • Orange icon: processing.

The tray menu provides Show Settings, a stateful recording action, and Quit.

X11

App-owned keyboard shortcuts

On pure X11 sessions, hotkey_backend = "auto" resolves to QuillSpeak's X11 passive-grab backend. Shortcut pages show both keyboard and Linux signal trigger controls. 

[general]
hotkey_backend = "auto"

[[shortcuts]]
id = "default"
trigger = { type = "keyboard", accelerator = "Ctrl+Alt+Space" }

Keyboard hotkey down starts recording for that shortcut. Hotkey up stops the same shortcut and sends the captured audio to the transcription worker. Hotkey backends use explicit start/stop edges, not a blind toggle.

Output Pipeline

Each shortcut owns its model, language, speaker-mute preference, script transform, clipboard behavior, and paste shortcut. 

transcript
  -> optional script transform
  -> final text
  -> optional clipboard copy
  -> optional paste shortcut

If a script is enabled, its stdout becomes the final text. The original transcript is not copied as a fallback after script errors.

Development

Build from source

sudo apt install build-essential pkg-config cmake clang libclang-dev \
  libasound2-dev libpulse-dev libpipewire-0.3-dev \
  libgtk-4-dev libadwaita-1-dev

cargo run -p quillspeak --bin quillspeak
QUILLSPEAK_DEV_LOG=1 cargo run -p quillspeak --bin quillspeak

Release automation

Tag-driven deployment

Pushing a tag such as v0.0.1 runs the release workflow, builds the primary Debian packages, uploads checksums and a release manifest to GitHub Releases, and then the Pages workflow refreshes the Downloads section from the latest release. 

git tag v0.0.1
git push origin v0.0.1
